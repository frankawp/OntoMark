## 获取wiki实体的方法

方法一：直接从wiki/ 目录中通过目录和文件名获取。 如果需要了解具体实体的内容可以读取文件内容了解。

方法二：直接读取 .ontomark/index.json

方法三：调用ontomark index-query . "老张"

```
 索引文件位置
  .ontomark/index.json

  索引结构

  {
    "entities": {
      "张三": {
        "canonical": "张三",
        "type": "Person",
        "path": "Person/张三.md",
        "aliases": ["老张", "Zhang San"]
      },
      "Acme公司": {
        "canonical": "Acme公司",
        "type": "Organization",
        "path": "Organization/Acme公司.md",
        "aliases": ["Acme", "阿克米"]
      }
    },
    "aliases": {
      "老张": "张三",
      "Zhang San": "张三",
      "Acme": "Acme公司",
      "阿克米": "Acme公司"
    }
  }

  ---
  索引内容

  ┌──────────┬────────────────────────────────────────┐
  │   字段   │                  用途                  │
  ├──────────┼────────────────────────────────────────┤
  │ entities │ 以规范名称为 key，存储实体元信息       │
  ├──────────┼────────────────────────────────────────┤
  │ aliases  │ 以别名为 key，指向规范名称（反向索引） │
  └──────────┴────────────────────────────────────────┘

  每个实体记录：
  - canonical — 规范名称
  - type — 实体类型
  - path — 相对于 wiki/ 的文件路径
  - aliases — 别名列表

  ---
  索引用途

  快速查询实体是否存在（无需扫描所有 wiki 文件）：

  ontomark index-query . "老张"
  # → { found: true, canonical: "张三", type: "Person", path: "Person/张三.md" }

  匹配优先级（index-query 实现）：

  ┌────────┬──────────────────────────┬──────┐
  │ 优先级 │         匹配方式         │ 分数 │
  ├────────┼──────────────────────────┼──────┤
  │ 1      │ 规范名称完全匹配         │ 3    │
  ├────────┼──────────────────────────┼──────┤
  │ 2      │ 别名完全匹配             │ 2    │
  ├────────┼──────────────────────────┼──────┤
  │ 2.5    │ 大小写不同的完全匹配     │ 2.5  │
  ├────────┼──────────────────────────┼──────┤
  │ 1.5    │ 前缀匹配（张 匹配 张三） │ 1.5  │
  ├────────┼──────────────────────────┼──────┤
  │ 1      │ 包含匹配                 │ 1    │
  ├────────┼──────────────────────────┼──────┤
  │ 0.8    │ 别名模糊匹配             │ 0.8  │
  └────────┴──────────────────────────┴──────┘

  ---

```
