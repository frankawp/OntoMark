# /ontomark-query

查询 wiki 知识，生成回答，用户确认后可选存储。

## 工作流程

### 第一步：解析问题

1. 分析用户问题，识别涉及的实体/概念

### 第二步：查询实体

2. 调用 `ontomark index-query <project-path> <name>` 或带 `--fuzzy` 模糊匹配
3. Read → 读取相关实体页面

### 第三步：生成回答

4. 综合多个实体信息，生成回答
5. 根据问题类型选择输出格式

### 第四步：展示与存储

6. 向用户展示回答
7. 询问："这个回答有价值，要存入 wiki 作为 Topic 页面吗？"
   - 用户同意 → 调用 `ontomark wiki-write` → 创建 Topic 页面
   - 用户拒绝 → 结束，不存储

## 回答形式

| 问题类型 | 输出格式 | 示例 |
|---------|---------|------|
| 简单查询 | 直接文本 + 来源 | "John Doe 是一名工程师，来自 [[Article]]" |
| 对比分析 | Markdown 表格 | `\| 属性 \| A \| B \|` |
| 关系梳理 | Mermaid 图表 | `graph LR; A --> B` |
| 事件梳理 | 时间线列表 | `- 2026-01: 事件A` |

## 来源追溯

回答中每个关键信息必须标注来源：
- 单来源：`[[实体名]] (来源段落)`
- 多来源：`[[实体A]], [[实体B]]`
