# 知识库本体定义

> 为你的 wiki 定义知识抽取的维度和粒度。
> 从下方的框架中选择你需要的维度，命名、配置后即可使用。

## 知识维度

### Actor / 行动者

> 谁在行动？个人、组织、系统等。

- **粒度**：标准
- **属性**：
  - 角色/职位
  - 所属组织
  - 关键行动

### Event / 事件

> 发生了什么？里程碑、会议、发布、事故等。

- **粒度**：标准
- **属性**：
  - 时间
  - 地点
  - 参与方
  - 结果/影响

### Thing / 事物

> 被讨论或使用的对象？产品、概念、文档、技术等。

- **粒度**：标准
- **属性**：
  - 类型
  - 用途/功能
  - 关联实体

### Place / 地点

> 在哪里？地理位置、组织、虚拟空间等。

- **粒度**：粗略
- **属性**：
  - 类型（城市/国家/虚拟）

---

## 维度关联

维度之间的自然关联方式（AI 据此推断实体关系）：

- Actor **参与/发起** Event
- Actor **使用/创造** Thing
- Event **发生在** Place
- Thing **属于** 某个 Actor
- Actor **关联** 其他 Actor

---

## 模板使用说明

1. **首次 init**：AI 扫描输入目录 → 建议骨架 → 用户确认/修改 → 写入 ontology.md
2. **再次 init**：读取现有 ontology.md → 展示当前维度 → 用户调整
3. 粒度影响 AI 提取深度，但不影响数据结构
4. 维度关联是提示性的，AI 会根据上下文灵活判断
